# 03. 交互与视觉设计

## 设计方向

Notch Pilot 应该像一个安静的系统组件，而不是一个花哨桌面装饰。

关键词：

- 轻量
- 克制
- 高密度
- 可快速扫视
- 不打断当前任务

## 视觉原则

- 使用深色半透明背景，跟随系统外观。
- 默认黑色或深灰胶囊，与真实刘海自然融合。
- 紧凑态顶部贴近菜单栏内侧，不要像悬浮卡片一样“凸”出来。
- 圆角使用 Apple 风格连续圆角，展开态比紧凑态更柔和。
- 动画短而快，避免大幅弹跳。
- 图标优先，文字只在需要辨识时出现。
- 面板宽度有限，内容必须可扫视。

## 状态模型

```text
Hidden
  ↓ 启用并定位
Compact
  ↓ hover / click / hotkey / drag enter
Expanded
  ↓ mouse leave / esc / click outside / hotkey
Compact
```

## 紧凑态布局

```text
┌──────────────────────────┐
│  Pomodoro   14:26   Todo │
└──────────────────────────┘
```

规则：

- 宽度默认 220 px，可调整。
- 高度默认 34 px，可调整，紧凑态默认视觉高度更轻。
- 中间放主状态，左右放辅助状态。
- 如果屏幕有真实刘海，小岛与刘海宽度对齐或略宽。
- 如果没有刘海，居中显示，形态更像顶部控制柄。

## 展开态布局

```text
┌──────────────────────────────────────┐
│  14:26 Fri        Search / Command   │
├──────────────────────────────────────┤
│  [Todo] [Timer] [Apps] [Files] [AI]  │
├──────────────────────────────────────┤
│                                      │
│  当前模块内容                         │
│                                      │
├──────────────────────────────────────┤
│  Settings                  Collapse  │
└──────────────────────────────────────┘
```

规则：

- 默认宽度 520 px，高度 360 px。
- 面板顶部贴近菜单栏区域，视觉上像从状态栏延展出来。
- 内容超过高度时内部滚动，不撑大窗口。
- 拖拽文件进入时自动切到文件暂存模块。

## 模块交互

### Todo

- 输入框始终在顶部。
- 回车新增。
- 点击圆点完成。
- 完成项淡化并移到底部。

### 番茄钟

- 大号倒计时。
- 主按钮为开始/暂停。
- 次级按钮为重置。
- 紧凑态显示剩余时间。

### 快捷启动

- 使用 App 图标网格。
- 支持拖动排序。
- 右键打开删除菜单。

### 文件暂存

- 空状态只显示一个拖拽落点。
- 拖入时高亮边框。
- 文件行展示图标、名称、大小。
- 支持拖出到其他 App。

### AI

- 单轮快速问答为主。
- 不把它做成完整聊天软件。
- 历史默认本地短期保存，可关闭。

## 可访问性

- 所有按钮提供 accessibility label。
- 支持键盘 Tab 导航。
- 支持 Esc 收起。
- 支持 Reduce Motion，关闭复杂动画。
- 颜色对比度满足常规阅读。

## 设计边界

- 不复刻 iPhone Dynamic Island 动效。
- 不使用竞品图标和截图。
- 不把顶部区域做成广告入口。
- 不自动遮挡用户菜单栏图标。
