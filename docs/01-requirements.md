# 01. 需求分析

## 能不能做

能做。推荐做成原生 macOS 应用，而不是网页壳：

- SwiftUI 负责设置页、模块面板和常规界面。
- AppKit 负责顶部悬浮岛、无边框窗口、多显示器定位、拖拽、窗口层级。
- 本地数据使用 SwiftData 或 SQLite。
- AI、天气、歌词等外部能力做成可选插件。

不能做或不建议做的部分：

- 不破解竞品内购。
- 不复制竞品名称、图标、视觉素材、动效细节和商业文案。
- 不默认使用私有 macOS API，以免稳定性和分发风险过高。

## 产品定位

Notch Pilot 是一款 macOS 顶部效率岛：

- 在有刘海的 MacBook 上贴合真实刘海区域。
- 在无刘海外接显示器或 iMac/Mac mini 上提供模拟小岛。
- 通过紧凑态展示状态，通过展开态承载工具。
- 主打本地优先、轻量、可关闭、可扩展。

## 目标用户

- MacBook Pro / MacBook Air 刘海屏用户。
- 使用外接屏但想要顶部快捷工具的 Mac 用户。
- 不想为简单效率工具反复内购或订阅的用户。
- 常用 AI、Todo、番茄钟、快捷启动、临时文件中转的办公用户。

## 核心场景

### 场景 1：快速查看状态

用户正在写代码或文档，不想切换窗口，只看顶部小岛即可获取：

- 当前时间、日期。
- 天气温度。
- 今日进度、周进度。
- 当前番茄钟状态。
- 正在播放的音乐信息。

### 场景 2：快速执行动作

用户鼠标移到顶部小岛，展开后可以：

- 打开常用 App。
- 触发系统快捷指令。
- 新建 Todo。
- 开始或暂停番茄钟。
- 打开 AI 小窗提问。

### 场景 3：临时收纳

用户拖一个文件到顶部小岛：

- 文件进入临时文件架。
- 可从文件架拖出到 Finder、聊天工具或邮件。
- 可一键清空。

### 场景 4：轻量记录

用户按快捷键或点击小岛：

- 快速写一条速记。
- 转成 Todo。
- 保存在本地。

## 功能优先级

### P0 MVP 必须做

- 顶部小岛窗口：紧凑态、展开态、自动定位、开机启动选项。
- 基础状态：时间、日期、系统外观适配。
- Todo：新增、完成、删除、本地保存。
- 番茄钟：开始、暂停、完成提醒。
- 快捷启动：配置常用 App 并启动。
- 文件暂存区：拖入、展示、拖出、清空。
- 设置页：模块开关、宽度、高度、位置、展开方式。

### P1 强烈建议

- 天气：用户配置天气 API Key 或使用可替换 Provider。
- AI 对话：用户配置 OpenAI/DeepSeek/Qwen 等兼容接口。
- 速记：快速记录、搜索、转 Todo。
- 快捷键：展开/收起、快速记录、打开启动台。
- 多显示器支持：每个屏幕独立启用或只在主屏显示。

### P2 后续增强

- 音乐信息与歌词。
- 日历事件。
- 摄像头镜子。
- 系统面板：电池、网络、内存、CPU。
- 插件市场或本地插件目录。
- 主题编辑器。

## 非功能需求

- 启动后常驻内存尽量低于 120MB。
- 空闲 CPU 占用接近 0。
- 展开动画 60 FPS，低端机器可关闭动画。
- 不抢焦点，不影响菜单栏点击。
- 权限申请必须按功能触发，而不是首次启动一次性索取。
- 所有模块都可关闭。

## 成功标准

- 用户可以在 10 秒内理解小岛如何展开和收起。
- 80% 常用能力不依赖外部网络。
- 没有强制登录、强制内购、强制订阅。
- 即使关闭 AI、天气、歌词，核心效率功能仍完整可用。
