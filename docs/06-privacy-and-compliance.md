# 06. 隐私、权限与合规边界

## 基本原则

- 默认本地运行。
- 默认不联网。
- 默认不收集分析数据。
- 默认不读取用户文件内容。
- 权限按模块申请。
- 用户可以随时关闭模块和清空数据。

## 数据存储

本地保存：

- Todo。
- 番茄钟配置和历史。
- 快捷启动配置。
- 文件暂存引用。
- 速记。
- 外观设置。

Keychain 保存：

- AI API Key。
- 天气 API Key。

不应保存：

- 用户系统密码。
- 竞品授权信息。
- 未经用户选择的文件内容。
- AI 请求中的隐私上下文。

## 网络行为

无网络也应可使用：

- 顶部小岛。
- 时间日期。
- Todo。
- 番茄钟。
- 快捷启动。
- 文件暂存。
- 速记。

需要网络：

- 天气。
- AI。
- 后续可能的更新检查。

所有网络模块必须：

- 在设置中显示 Provider。
- 可关闭。
- 显示错误状态。
- 不静默上传本地数据。

## 权限说明

### 通知权限

用途：

- 番茄钟完成提醒。
- Todo 到期提醒。

### 文件访问权限

用途：

- 文件暂存区记录用户拖入的文件。

实现：

- 使用安全书签。
- 不主动扫描用户目录。

### 摄像头权限

用途：

- 后续镜子模块。

策略：

- P2 再做。
- 用户启用模块时再申请。

### 辅助功能权限

用途：

- 复杂快捷键或自动化控制。

策略：

- MVP 尽量不依赖。
- 只有明确功能需要时申请。

### 自动化权限

用途：

- 与 Music、Spotify、Shortcuts 等 App 交互。

策略：

- P2 再做。
- 每个集成独立说明用途。

## 合规边界

允许：

- 做同类功能的原创实现。
- 使用公开 API。
- 让用户配置自己的第三方 API Key。
- 提供本地可用的免费核心功能。

不允许：

- 破解或绕过 Nook X / PopX / ShoX / NotchNook 内购。
- 复制竞品图标、截图、动效、文案和品牌。
- 未授权读取其他 App 私有数据。
- 在用户不知情时上传文件、笔记、Todo 或屏幕内容。

## 分发策略

优先顺序：

1. 本地开发自用。
2. 签名 + Notarized DMG 分发。
3. 评估 App Store 上架。

如果要上架 App Store，需要额外评估：

- 沙盒限制。
- 私有 API 风险。
- 第三方 App 控制能力。
- AI 内容和隐私声明。
